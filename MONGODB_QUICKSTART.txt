📋 SETUP CHECKLIST

  ☐ 1. Create MongoDB Atlas account
       → https://www.mongodb.com/cloud/atlas
  
  ☐ 2. Create a free M0 cluster (512MB)
       → Choose AWS/GCP/Azure + nearest region
  
  ☐ 3. Create database user
       → Database Access → Add New User
       → Save username & password!
  
  ☐ 4. Configure network access
       → Network Access → Add IP Address
       → Allow 0.0.0.0/0 (for development)
  
  ☐ 5. Get connection string
       → Database → Connect → Connect your application
       → Copy the connection string
  
  ☐ 6. Update .env.local file
       → Replace <username>, <password>, <cluster-url>
  
  ☐ 7. Seed the database
       → npm run seed
  
  ☐ 8. Start your app
       → npm run dev

📝 ENVIRONMENT VARIABLE (.env.local)

MONGODB_URI=mongodb+srv://USERNAME:PASSWORD@cluster0.xxxxx.mongodb.net/?retryWrites=true&w=majority&appName=ecom-app

Replace:
  • USERNAME → Your database user
  • PASSWORD → Your database password  
  • cluster0.xxxxx → Your cluster URL

⚠️  IMPORTANT: No < > brackets in the actual URI!

🔍 VERIFICATION STEPS

  1. Start dev server (npm run dev)
  2. Check terminal for errors
  3. Open http://localhost:3000
  4. Verify products are displayed
  5. Test admin panel at /admin
  6. Check MongoDB Atlas → Browse Collections

🐛 COMMON ISSUES

  Issue: "Invalid/Missing environment variable"
  → Check .env.local exists and MONGODB_URI is set

  Issue: "bad auth" error
  → Verify username & password in connection string

  Issue: "connection refused"  
  → Check Network Access in MongoDB Atlas

  Issue: No products showing
  → Run: npm run seed

  • MongoDB Compass: Desktop GUI for database management
  • Check MongoDB Atlas metrics for performance insights
